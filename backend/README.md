# Backend — Kutu Digitizer API

**Hono + TypeScript + Drizzle + Postgres + Better Auth · runs on :4000**

---

## Initialize (Saturday morning)

```bash
cd backend
npm init -y
# Install per /TECH-STACK.md Section 8 → Backend
```

## Expected Structure (after init)

```
backend/
├── package.json
├── tsconfig.json
├── drizzle.config.ts
├── Dockerfile
├── .env.local         (gitignored)
├── .env.prod          (EC2 only, gitignored)
├── .env.example
└── src/
    ├── index.ts                    # Hono entry · binds :4000
    ├── env.ts                      # @t3-oss/env-core schema
    ├── auth/
    │   └── index.ts                # Better Auth config
    ├── db/
    │   ├── client.ts               # Drizzle client
    │   ├── schema.ts               # Tables (users, sessions, tabung, ...)
    │   └── migrations/             # Auto-generated by drizzle-kit
    ├── routes/
    │   ├── auth.ts                 # Better Auth handler mount
    │   ├── tabung.ts               # CRUD tabung
    │   ├── members.ts              # Invite + join
    │   ├── contributions.ts        # TNG flow + ledger
    │   ├── rotations.ts            # Rotation logic
    │   ├── penasihat.ts            # Claude streaming chat
    │   ├── uploads.ts              # S3 presigned URLs
    │   └── webhooks/
    │       └── tng.ts              # TNG payment webhooks
    ├── services/
    │   ├── tng.ts                  # TNG client wrapper
    │   ├── s3.ts                   # AWS S3 client + presigner
    │   ├── claude.ts               # Anthropic SDK wrapper
    │   └── rotation-engine.ts      # Cycle math + scheduling
    ├── jobs/
    │   └── monthly-rotation.ts     # node-cron scheduled task
    ├── middleware/
    │   ├── logger.ts               # pino-hono
    │   └── rate-limit.ts           # hono-rate-limiter
    └── lib/
        ├── logger.ts               # pino instance
        └── errors.ts               # error classes + handler
```
